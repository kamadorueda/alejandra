# Frontend Modernization Plan

## Overview
Migrate the existing Alejandra frontend from a Create React App + Tachyons setup to a modern stack based on kamadorueda.github.io architecture, while preserving all existing functionality. The new frontend will be developed in an orphaned `web` branch.

### Tooling Approach
- **Match**: kamadorueda.github.io's core build stack (Vite, TypeScript, Tailwind CSS, React 19, pnpm)
- **Exclude**: Formatters, linters, tests (not needed at this stage)
- **Result**: Lean, modern setup focused on functionality

## Current State Analysis

### Existing Frontend
- **Build tool**: Create React App (react-scripts)
- **Styling**: Tachyons CSS utility library
- **Runtime**: React (18+) with vanilla JS
- **Key dependencies**: codemirror, react-diff-viewer, alejandra_front (WASM)
- **Main features**:
  - Side-by-side code diff view (before/after formatting)
  - CodeMirror editor for Nix code input
  - Random file fetching from Nixpkgs
  - Permalink sharing (URL-based state)
  - WASM formatter integration

### Reference Architecture (kamadorueda.github.io)
- **Build tool**: Vite
- **Styling**: Tailwind CSS (v4) with PostCSS
- **Language**: TypeScript
- **UI Framework**: React Router (v7), Headless UI
- **Testing**: Vitest + React Testing Library
- **Linting/Formatting**: ESLint + Prettier (but not required for this migration)
- **Modern features**: Native ES modules, faster HMR, optimized production builds

## WASM Module Strategy

### Assumption
The WASM package (`alejandra_front`) is available via `file:./pkg` reference. This can be:
1. **Copied locally** from the main repository's build output
2. **Built from source** in the Nix environment
3. **Installed from a published package** (future enhancement)

Current implementation uses local file dependency for development simplicity.

## Important Considerations

### CodeMirror v5 Stability
CodeMirror 5 is intentionally kept stable. Migration to v6 is a future enhancement that would require significant refactoring of the editor integration.

### WASM Integration
The WASM module path is configurable. Current setup uses a local file dependency, but can be updated to reference a published npm package when available.

### No Build Tools for Code
Intentionally excluded formatters, linters, and test frameworks at this stage per requirements. These can be added in a future phase if needed.

### Tailwind CSS v4
Uses the new `@theme` syntax for custom variables, which differs from v3. Configuration is minimal as v4 includes most utilities by default.

### React Router Not Needed
Unlike the reference project, this simple single-page formatter doesn't require React Router. It can be added if routing is needed in the future.
